A [SharedTree](./index.mdx)'s data is organized into a tree of nodes.
See [node types](./node-types.mdx) for details on the available node types.

## Node Utilities

### `Tree.key`

```typescript
Tree.key(node: SharedTreeNode): number | string
```

Returns the field key that `node` is stored under.
Returns a string in all cases, except for array nodes where it returns the numeric index.

### `Tree.parent`

```typescript
Tree.parent(node: SharedTreeNode): SharedTreeNode
```

Returns the parent node of `node`. For example, to delete a note only if it belongs to a group's notes array or is a direct root child, get the parent and test its type:

```typescript
const parent = Tree.parent(note);

if (Tree.is(parent, Notes) || Tree.is(parent, Items)) {
	const index = parent.indexOf(note);
	parent.removeAt(index);
}
```

### `Tree.status`

```typescript
Tree.status(node: SharedTreeNode): TreeStatus
```

Returns the current status of `node`. Possible values are:

-   **New**: Created but not yet inserted into the tree.
-   **InDocument**: Parented (directly or indirectly) under the root field.
-   **Removed**: Not under the root field, but may still be restorable by this or other clients.
-   **Deleted**: Cannot be restored by this client, though other clients may still restore it.

### `Tree.schema`

```typescript
Tree.schema(node: SharedTreeNode): TreeNodeSchema
```

Returns the schema object for `node`.

### `Tree.is`

Use `Tree.is()` to test a node's type when its type is unknown.

```typescript
Tree.is(someNode: SharedTreeNode, nodeType: TreeNodeSchema | T): boolean
```

Returns `true` if `someNode` is of type `nodeType`, where `T` is a class derived from a `SchemaFactory` method (`object()`, `map()`, or `array()`).

```typescript
if (Tree.is(myNode, Note)) {
	// Process Note nodes.
}
```

See also the `Tree.parent()` example above.
