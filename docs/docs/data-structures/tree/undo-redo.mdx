---
title: Undo Redo Support
sidebar_position: 7
---

`SharedTree` makes undo/redo stacks straightforward. Listen for `commitApplied` on `TreeView` to receive a `Revertible` object from each `Commit`.

Commits come in three kinds:

-   **Default**: a normal local commit — goes on the undo stack.
-   **Undo**: produced by reverting a Default or Redo `Revertible` — goes on the redo stack.
-   **Redo**: produced by reverting an Undo `Revertible` — goes on the undo stack.

To undo a change, call `revert()` on the `Revertible`. This restores the properties last changed by the local client to their previous state, overwriting any intervening changes from other clients. For example, if the local client moved 3 items into an array and a remote client then moved one of those items elsewhere, reverting the local change returns that item to its original position.

See the [Shared Tree Demo](https://github.com/microsoft/FluidExamples/tree/main/brainstorm/src/utils/undo.ts) for a working example.
