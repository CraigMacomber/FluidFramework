The `TreeView` object and its children expose properties and methods for reading and editing the tree.
The APIs mirror standard TypeScript/JavaScript syntax for primitives, objects, maps, and arrays, with some differences in editing APIs to make merge semantics explicit.

## Leaf Node APIs

Leaf nodes are read and written using dot notation and the assignment operator (`=`), exactly like JavaScript primitives:

```typescript
myNewsPaperTree.articles[1].headline = "Man bites dog";
```

Values returned from leaf nodes are JavaScript primitive types, not schema types (e.g., `number`, not `sf.number`). `FluidHandle` is the exception — it is returned as-is.

```typescript
const pointsForDetroitTigers: number = seasonTree.tigersTeam.game1.points;
```

## Object Node APIs

### Reading Object Properties

Read object nodes exactly as you would a JavaScript object:

```typescript
const pointsForDetroitTigers: number = seasonTree.tigersTeam.game1.points;

const counterHandle: FluidHandle = myTree.myObjectNode.myHandle;

const myItems: Items = stickyNotesTree.items;
```

### Creating Objects

Create objects using the constructor of the class derived from `SchemaFactory.object()`:

```typescript
const babyShowerNote = new Note({
    id: Guid.create().toString(),
    text: "Baby shower is at 3 PM today.",
    author: "Bob",
    lastChanged: 19697 // Days since January 1, 1970, the Unix epoch.
    votes: ["0"]
});
```

See [Array node APIs](#array-node-apis) for how to insert this note into the tree.

### Editing Object Properties

Assign a new value using `=`.
See [object merge semantics](https://github.com/microsoft/FluidFramework/blob/main/packages/dds/tree/docs/user-facing/object-merge-semantics.md) for details.

```typescript
rectangle.topLeft = new Point({ x: 0, y: 0 });
```

```typescript
babyShowerNote.author = "The Joneses";
```

Optional properties can be cleared by assigning `undefined` to them.

```typescript
proposal.text = undefined;
```

If the new value is a node (not a primitive), its [status](./nodes.mdx#treestatus) must be `TreeStatus.New`. A node that was previously inserted cannot be re-set, even if it has since been removed.

## Map Node APIs

### Map Node Read APIs

Map node read APIs match the JavaScript `Map` API:

```typescript
has(key): boolean
```

Returns `true` if the key is present.

```typescript
get(key): T | undefined
```

Returns the value for the key, or `undefined` if absent.

```typescript
keys(): IterableIterator<string>
```

Returns an iterator over the map's keys.

```typescript
values(): IterableIterator<T>
```

Returns an iterator over the map's values.

```typescript
entries(): IterableIterator<[string, T]>
```

Returns an iterator over key/value pairs.

```typescript
map(callback: ()=>[]): IterableIterator<[string, T]>
```

Returns a plain JavaScript array (not a map or array node) produced by applying the callback to each entry, like [Array.map](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array/map).

### Map Node Write APIs

Map node write APIs are similar to JavaScript `Map`.
See [map merge semantics](https://github.com/microsoft/FluidFramework/blob/main/packages/dds/tree/docs/user-facing/map-merge-semantics.md) for details.

```typescript
set(key: string, value: T)
```

Sets or updates the value for the specified key, adding it if absent. Note:

-   `T` must conform to the map's schema (e.g., `number` or `string` for `class MyMap extends sf.map([sf.number, sf.string])`).
-   If `value` is a node, its [status](./nodes.mdx#treestatus) must be `TreeStatus.New`. A previously-inserted node cannot be re-set.
-   If multiple clients set the same key simultaneously, the last applied edit wins. See [Types of distributed data structures](../overview.mdx) for the meaning of "simultaneously".

```typescript
delete(key: string): void
```

Removes the item with the specified key. If one client sets a key while another deletes it simultaneously, the key is deleted only if the delete op is applied last. See [Types of distributed data structures](../overview.mdx).

### Map Node Properties

```typescript
size: number;
```

The number of entries in the map.

## Array Node APIs

### Array Node Read APIs

Array nodes support all non-mutating JavaScript [Array](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array) methods. See [copying vs. mutating methods](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array#copying_methods_and_mutating_methods). Methods that return an array (e.g., `Array.map()`) return a plain JavaScript array, not an array node — so calling `.map()` on a `Notes` node returns `Array`, not `Notes`.

### Array Node Write APIs

Array node write APIs differ from JavaScript arrays to better support collaborative editing. There are three categories: Insert, Remove, and Move.
See [array merge semantics](https://github.com/microsoft/FluidFramework/blob/main/packages/dds/tree/docs/user-facing/array-merge-semantics.md) for details.

#### Insert Methods

The three insert methods share these constraints:

-   `T` must conform to the array's schema (e.g., `number` or `string` for `class MyArray extends sf.array([sf.number, sf.string])`).
-   Inserted nodes must have [status](./nodes.mdx#treestatus) `TreeStatus.New`. A previously-inserted node cannot be re-inserted.

```typescript
insertAt(index: number, value: Iterable<T>)
```

Inserts value(s) at `index`. If `index` exceeds the array length, items are appended.

```typescript
insertAtStart(value: Iterable<T>)
```

Inserts value(s) at the start. Equivalent to `insertAt(0, …)`.

```typescript
insertAtEnd(value: Iterable<T>)
```

Inserts value(s) at the end. Equivalent to `insertAt(Infinity, …)`.

#### Remove Methods

The two remove methods share these behaviors:

-   Removed items are retained internally to support undo.
-   A concurrent move operation can resurrect a removed item. For example, if one client removes items 3–5 while another moves items 4 and 5, and the move is ordered last, only item 3 is removed (4 and 5 are moved to their destination). If the remove is ordered last, all three are removed regardless of where they reside.
-   Removes never suppress concurrent inserts. For example, removing items 10–15 while another client inserts at index 12 removes the original 10–15, but the inserted item lands between the former item 9 and the former item 16, regardless of operation order.

For the meaning of "simultaneously", see [Types of distributed data structures](../overview.mdx).

```typescript
removeAt(index: number)
```

Removes the item at the given `index`.

```typescript
removeRange(start?: number, end?: number)
```

Removes items from `start` (inclusive) to `end` (exclusive). If `end` is omitted, removes from `start` to the end. If `start` is omitted, it defaults to 0. Calling `removeRange()` with no arguments removes all items.

#### Move Methods

The three move methods can move items within an array or between arrays; when moving between arrays, call the method on the destination array. `T` is any class derived from `SchemaFactory.array()` (e.g., `Notes` or `Items`).

These methods share the following behaviors:

-   If multiple clients move the same item simultaneously, the last edit applied determines the final destination.
-   A concurrent remove can affect moved items. For example, if one client moves items 3–5 while another removes items 4 and 5, and the remove is ordered last, items 4 and 5 are removed from their destination. If the move is ordered last, all three items reach the destination.
-   Moved nodes must have [status](./nodes.mdx#treestatus) `TreeStatus.InDocument`.
-   Single-item variants of each method are also available.

For the meaning of "simultaneously", see [Types of distributed data structures](../overview.mdx).

```typescript
moveRangeToStart(sourceStartIndex: number, sourceEndIndex: number, source?: T)
```

Moves the specified items to the start of the array. Pass `source` when moving from a different array.

```typescript
moveRangeToEnd(sourceStartIndex: number, sourceEndIndex: number, source?: T)
```

Moves the specified items to the end of the array. Pass `source` when moving from a different array.

```typescript
moveRangeToIndex(destinationGap: number, sourceStartIndex: number, sourceEndIndex: number, source?: T)
```

Moves the items to `destinationGap` in the destination array. Pass `source` when moving from a different array.
When moving within the same array, `destinationGap` is interpreted as if the moved items were not removed first (i.e., as if inserting a copy at the gap position).
See [the API docs](../../api/fluid-framework/treearraynode-interface.md#moverangetoindex-methodsignature) for details.
